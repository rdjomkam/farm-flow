# BUG-025 — Site model missing `isPlatform` flag
**Sévérité :** Haute
**Détecté par :** @project-manager
**Sprint :** 7 (Multi-tenancy)
**Fichier(s) :** prisma/schema.prisma, src/types/models.ts

## Description
Il n'existe aucun moyen d'identifier le site plateforme (DKFarm) parmi les sites.
Le modèle `Site` n'a pas de champ `isPlatform`. Actuellement, la distinction repose
uniquement sur le rôle global `ADMIN` de l'utilisateur, ce qui est insuffisant.

## Étapes de reproduction
1. Regarder le modèle Site dans schema.prisma → aucun champ `isPlatform`
2. Impossible de savoir programmatiquement quel site est la plateforme

## Cause racine
Le concept de "site plateforme unique" n'a pas été modélisé lors du Sprint 7 (multi-tenancy).

## Fix
- [ ] Ajouter `isPlatform Boolean @default(false)` sur le modèle `Site` dans `prisma/schema.prisma`
- [ ] Ajouter contrainte d'unicité partielle : un seul site peut avoir `isPlatform: true` (via `@@unique` conditionnel ou validation API)
- [ ] Mettre à jour l'interface TypeScript miroir dans `src/types/models.ts`
- [ ] Générer et appliquer la migration
- [ ] Mettre à jour `prisma/seed.sql` : le site DKFarm existant doit avoir `isPlatform: true`
- [ ] Créer un helper `getPlatformSite()` dans `src/lib/queries/sites.ts`
- [ ] Valider côté API que seul 1 site peut être `isPlatform: true`
- [ ] Test de non-régression ajouté
- [ ] Tous les tests passent
- [ ] Build OK
