# BUG-030 — ~130 hardcoded French strings not translated (i18n incomplete)
**Sévérité :** Moyenne
**Détecté par :** @project-manager
**Sprint :** 40-42 (i18n)
**Fichier(s) :** Multiples (~25 fichiers)

## Description
Malgré les sprints i18n (39-42), environ 130 chaînes de caractères en français restent hardcodées
dans le code au lieu d'utiliser `useTranslations()` / `getTranslations()` de next-intl.
Les composants concernés n'ont aucun appel à `t()` ou mélangent chaînes traduites et hardcodées.

## Cause racine
Les sprints i18n (39-42) ont couvert les couches core, vagues, relevés, stock, ventes, users,
alevins et commissions, mais ont omis les modules plus récents (remises, packs, activations,
planning, alertes) et les composants UI transversaux.

## Fix
- [ ] Créer les fichiers de traduction manquants (fr + en) pour chaque namespace
- [ ] Pour chaque fichier listé : ajouter `useTranslations()` et remplacer les chaînes hardcodées par `t("key")`
- [ ] Pour les Server Components : utiliser `getTranslations()` au lieu de `useTranslations()`
- [ ] Mettre à jour `src/messages/index.ts` pour inclure les nouveaux namespaces
- [ ] Tous les tests passent
- [ ] Build OK
