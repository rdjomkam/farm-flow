# BUG-016 ‚Äî Produits consommes non affiches dans le detail et la modification d'un releve

**Severite :** Haute
**Detecte par :** @ronald
**Sprint :** 8
**Assigne a :** @developer (UI + API) + @db-specialist (queries + logique stock)
**Fichier(s) :** voir section "Fichiers impactes"

## Description

Lors de la creation d'un releve (alimentation, qualite eau, mortalite), l'utilisateur peut associer des produits consommes (via `ConsommationFields`). Ces consommations sont bien enregistrees en base (`ReleveConsommation` + `MouvementStock`). Cependant :

1. **Affichage (detail releve)** : la liste des releves sur la page vague (`RelevesList`) ne montre pas les produits associes. L'utilisateur ne sait pas quels produits ont ete consommes pour un releve.

2. **Modification (dialog)** : le `ModifierReleveDialog` ne charge pas et n'affiche pas les consommations existantes. Si l'utilisateur modifie un releve, il ne voit pas les produits qui avaient ete associes a la creation.

## Etapes de reproduction

1. Creer un releve de type "Alimentation" avec un produit consomme (ex: 5kg de granule)
2. Aller sur la page de la vague > section releves
3. Observer que le releve ne montre pas le produit consomme
4. Cliquer sur "Modifier" (crayon) sur le releve
5. Observer que le dialog de modification ne montre pas les consommations

## Cause racine

### Affichage
- `getVagueById()` dans `src/lib/queries/vagues.ts` (ligne 25) fait `releves: { orderBy: { date: "desc" } }` mais n'inclut PAS `consommations` (ni la relation `produit` sur chaque consommation)
- `RelevesList` (composant) n'a pas de code pour afficher les consommations

### Modification
- `ModifierReleveDialog` ne charge pas les consommations du releve
- Le composant ne gere que les champs scalaires (poids, temperature, etc.)
- Le `buildBody()` n'envoie pas de consommations
- La route PUT `/api/releves/[id]` et `updateReleve()` ne gerent pas la modification des consommations

### GET API
- `getReleveById()` dans `src/lib/queries/releves.ts` (ligne 178-181) fait `include: { consommations: true }` mais n'inclut PAS la relation `produit` (nom, unite) sur chaque consommation ‚Äî on n'a que le `produitId` brut

### UI - Affichage
| Fichier | Modification |
|---------|-------------|
| `src/components/vagues/releves-list.tsx` | Afficher les produits consommes sous chaque releve (nom produit, quantite, unite) |

### UI - Modification
| Fichier | Modification |
|---------|-------------|
| `src/components/releves/modifier-releve-dialog.tsx` | Charger et afficher les consommations existantes. Reutiliser `ConsommationFields` pour permettre l'ajout/suppression de produits. Attention : la modification de consommations implique des mouvements de stock inverses (annulation de la sortie originale + nouvelle sortie) |

### Types
| Fichier | Modification |
|---------|-------------|
| `src/types/models.ts` | Verifier que `Releve` peut porter un champ optionnel `consommations?: ReleveConsommation[]` ou creer un type `ReleveWithConsommations` |

## Complexite

**Moyenne-Haute** ‚Äî La partie affichage est simple (inclure les relations + afficher). La partie modification est plus complexe car elle implique :
- Charger les consommations existantes dans le dialog
- Permettre l'ajout/suppression de lignes
- Calculer les deltas de stock (annuler les anciennes sorties, creer les nouvelles)
- Gerer les cas limites (stock insuffisant pour un nouvel ajout)
