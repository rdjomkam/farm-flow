# BUG-005 — Overflow horizontal et vertical sur la page detail vague
**Severite :** Moyenne
**Detecte par :** @project-manager
**Sprint :** 6
**Fichier(s) :** src/components/layout/app-shell.tsx, src/app/vagues/[id]/page.tsx

## Description
Sur la page `/vagues/[id]`, un overflow horizontal (scrollbar horizontale) et vertical apparait, surtout sur mobile (360px). La page empile beaucoup de contenu (info card, 5 indicateurs, chart, liste de releves, bouton retour) et plusieurs elements n'ont pas de controle d'overflow adequat.

Ce qui se passe : scrollbar horizontale visible sur mobile, contenu deborde du viewport 360px.
Ce qui devrait se passer : pas de scrollbar horizontale, tout le contenu reste dans le viewport.

## Etapes de reproduction
1. Ouvrir `/vagues/[id]` sur un ecran de 360px de large
2. Observer la scrollbar horizontale
3. Verifier le header quand la vague est EN_COURS (titre + bouton Releve + bouton Cloturer)

## Cause racine
1. `<main>` dans app-shell.tsx n'a pas de `overflow-x-hidden`, donc tout contenu debordant cree une scrollbar horizontale
2. Le header de la page vague contient trop d'elements sur une seule ligne en mobile (titre + SiteSelector + boutons + UserMenu)

## Fix
- [x] Fichier(s) modifie(s)
  - `src/components/layout/app-shell.tsx` — ajout `overflow-x-hidden` sur `<main>`
  - `src/app/vagues/[id]/page.tsx` — boutons d'action deplaces sous le header
