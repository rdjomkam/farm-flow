# BUG-009 — Page /stock crash: "Cannot read properties of undefined (reading 'findMany')"
**Severite :** Haute
**Detecte par :** @team-lead (signale par utilisateur)
**Sprint :** 8
**Fichier(s) :** src/lib/queries/produits.ts, fournisseurs.ts, mouvements.ts, commandes.ts, src/lib/db.ts

## Description
La page /stock affiche une erreur "Cannot read properties of undefined (reading 'findMany')". La page est completement inaccessible.

Ce qui se passe : crash avec erreur "Cannot read properties of undefined (reading 'findMany')"
Ce qui devrait se passer : la page /stock s'affiche normalement avec le dashboard stock.

## Etapes de reproduction
1. Naviguer vers /stock
2. Erreur immediate

## Cause racine
`getProduitsEnAlerte` dans `src/lib/queries/produits.ts` utilisait `prisma.produit.fields.seuilAlerte` (FieldRef) dans une clause `where` pour comparer deux colonnes. Cette syntaxe n'est pas supportee par l'adaptateur PrismaPg, ce qui faisait crasher le Promise.all de la page /stock.

## Fix applique par @developer
- [x] Diagnostique : `getProduitsEnAlerte` utilisait FieldRef non supporte par PrismaPg
- [x] Fix : recuperer les produits actifs avec `seuilAlerte > 0`, puis filtrer cote application avec `p.stockActuel <= p.seuilAlerte`
- [x] Client Prisma regenere (`npx prisma generate`)
- [x] Fichier(s) modifie(s) : `src/lib/queries/produits.ts`
- [x] Build OK
