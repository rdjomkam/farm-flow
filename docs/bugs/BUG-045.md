# BUG-045 — Bacs section affiche un `nombrePoissons` périmé (pas mis à jour par les mortalités)
**Sévérité :** Moyenne
**Détecté par :** @claude (inspection prod sur Vague 26-02)
**Sprint :** courant
**Fichier(s) :** src/components/pages/vague-detail-page.tsx, src/components/vagues/vague-bacs-section.tsx

## Description
La section Bacs de la page détail d'une vague affiche `nombrePoissons` (champ Prisma sur `AssignationBac`), qui n'est mis à jour que lors des comptages, calibrages et ventes. Les relevés de mortalité ne modifient pas ce champ. En conséquence, le nombre affiché par bac actif est potentiellement très supérieur au nombre réel de poissons vivants, trompant le pisciculteur.

Le serveur calcule déjà `vivantsByBac` (Map<bacId, vivants>) via `computeVivantsByBac` à la ligne 123 de `vague-detail-page.tsx`, mais cette map n'est pas transmise à `VagueBacsSection`.

## Étapes de reproduction
1. Ouvrir une vague avec des relevés de mortalité enregistrés.
2. Consulter la section "Bacs" de la page détail.
3. Observer que le nombre de poissons affiché ne correspond pas au nombre vivants réel.

## Cause racine
`vivantsByBac` est calculé côté serveur pour le graphique mais n'est pas transmis en prop à `VagueBacsSection`. Le composant utilise `a.nombrePoissons` (champ DB stale) au lieu de la valeur calculée dynamiquement.

## Fix
- [x] `src/components/pages/vague-detail-page.tsx` — enrichir `bacsActifs` avec `vivants` issus de `vivantsByBac` avant de passer la prop.
- [x] `src/components/vagues/vague-bacs-section.tsx` — mettre à jour le type de prop en `(AssignationBacForVague & { vivants: number | null })[]` et afficher "X actuels (Y au départ)" pour les bacs actifs.
- [x] `src/messages/fr/vagues.json` — ajouter la clé `bacsSection.poissonsActuels`.
- [x] `src/messages/en/vagues.json` — ajouter la clé `bacsSection.poissonsActuels`.
- [x] Test de non-régression ajouté (`src/components/vagues/__tests__/vague-bacs-section.test.tsx`).
- [x] Tous les tests passent (5 échecs pré-existants documentés dans docs/tests/rapport-bugs-041-042-043.md).
