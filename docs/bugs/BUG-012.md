# BUG-012 — Le lien "Permissions" dans le dialog membre ouvre une page separee au lieu de rester dans le dialog
**Severite :** Moyenne
**Detecte par :** @team-lead (signale par utilisateur)
**Sprint :** 8
**Fichier(s) :** src/components/sites/member-actions-dialog.tsx, src/components/sites/member-permissions-client.tsx

## Description
Dans le dialog de gestion membre (CR-008), le lien "Gerer les permissions" navigue vers `/sites/[id]/members/[userId]/permissions` — une page separee. L'utilisateur s'attend a ce que toutes les actions membre (role, permissions, retrait) soient gerees **dans le dialog**, conformement a l'esprit de CR-008.

Ce qui se passe : clic sur "Permissions" → navigation vers une nouvelle page, le dialog se ferme.
Ce qui devrait se passer : les permissions sont editables directement dans le dialog (section depliable avec checkboxes, ou sous-dialog).

## Etapes de reproduction
1. Ouvrir /sites/[id]
2. Cliquer sur le bouton Gerer d'un membre
3. Cliquer sur "Gerer les permissions"
4. Le dialog se ferme et on navigue vers une page separee

## Cause racine
CR-008 a implemente le lien permissions comme un `<Link>` vers la page existante au lieu d'integrer le composant de permissions directement dans le dialog.

## Fix suggere
Integrer le composant `member-permissions-client.tsx` (qui existe deja) directement dans le dialog `member-actions-dialog.tsx`, par exemple :
- Section depliable "Permissions" avec les checkboxes
- Ou vue multi-etapes dans le dialog (info → permissions)

- [x] Fichier(s) modifie(s) : member-actions-dialog.tsx, site-detail-client.tsx
- [x] Integrer les permissions dans le dialog au lieu de naviguer
