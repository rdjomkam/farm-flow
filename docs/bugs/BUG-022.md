# BUG-022 — Modules abonnements absents de SiteModule + pas de distinction platform/site-level
**Severite :** Moyenne
**Detecte par :** @project-manager
**Sprint :** 38
**Fichier(s) :** prisma/schema.prisma, src/types/models.ts, src/lib/permissions-constants.ts, src/components/sites/site-detail-client.tsx

## Description
Les modules ajoutes dans les Sprints 33-35 (Abonnement, Commissions, Remises) ne sont pas
enregistres dans l'enum `SiteModule` (Prisma + TypeScript). Ils sont gates uniquement par des
permissions via `PHASE3_MODULE_PERMISSIONS` dans la sidebar, mais ne participent pas au systeme
de modules activables par site.

De plus, l'enum `SiteModule` ne distingue pas les modules **site-level** (activables/desactivables
par site, comme Reproduction, Ventes) des modules **platform-level** (toujours disponibles pour
toutes les fermes, gates uniquement par les permissions, comme Abonnements, Commissions).

Actuellement, tous les modules de `MODULE_CONFIG` dans `site-detail-client.tsx` sont affiches
comme toggleables par l'admin du site, ce qui poserait probleme si on ajoute les modules
d'abonnement sans distinction : un admin pourrait desactiver les abonnements pour son site.

## Etapes de reproduction
1. Aller dans /settings/sites/[id] en tant qu'admin
2. Observer les modules toggleables — aucun module abonnement/commission/remise n'apparait
3. Dans la sidebar, les modules "Abonnement", "Portefeuille", etc. sont gates par `PHASE3_MODULE_PERMISSIONS` mais pas par `MODULE_LABEL_TO_SITE_MODULE`
4. Pas de moyen de distinguer un module platform d'un module site

## Cause racine
1. L'enum `SiteModule` n'a pas ete etendu lors des Sprints 30-35
2. Il n'existe aucun champ ou metadata pour distinguer les modules platform des modules site
3. `MODULE_CONFIG` dans `site-detail-client.tsx` affiche tous les `SiteModule` sans filtrage

## Fix
- [ ] Ajouter les valeurs `ABONNEMENTS`, `COMMISSIONS`, `REMISES` a l'enum `SiteModule` (Prisma + TS)
- [ ] Ajouter un champ `isPlatform` dans les metadata de module (soit dans l'enum avec convention, soit dans une config TS)
- [ ] Filtrer `MODULE_CONFIG` dans `site-detail-client.tsx` pour exclure les modules platform
- [ ] Ajouter les mappings dans `MODULE_LABEL_TO_SITE_MODULE` pour les modules abonnements
- [ ] Ne PAS gater la sidebar par `SiteModule` pour les modules platform (garder le gate par permission uniquement)
- [ ] Test de non-regression ajoute
- [ ] Tous les tests passent
- [ ] Build OK
