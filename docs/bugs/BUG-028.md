# BUG-028 — PACKS_PROVISIONING wrongly classified as site-level module
**Sévérité :** Moyenne
**Détecté par :** @project-manager
**Sprint :** 38 (SiteModule platform/site)
**Fichier(s) :** src/lib/site-modules-config.ts

## Description
Le module `PACKS_PROVISIONING` est classé comme `level: "site"` dans `SITE_MODULES_CONFIG`,
ce qui signifie qu'il est toggleable par un admin de site et apparaît dans la page de configuration
des modules d'un site client.

Or, le provisioning de packs est une fonctionnalité exclusivement plateforme : seul DKFarm
provisionne des packs vers les sites clients. Un site client ne devrait jamais voir ni toggler
ce module.

## Étapes de reproduction
1. Aller dans la page de configuration d'un site client → section "Modules"
2. Observer : "Packs & Provisioning" apparaît comme module toggleable

## Cause racine
Mauvaise classification lors de la création de `SITE_MODULES_CONFIG` dans le Sprint 38.

## Fix
- [ ] Modifier `src/lib/site-modules-config.ts` : changer `SiteModule.PACKS_PROVISIONING` de `level: "site"` à `level: "platform"`
- [ ] Vérifier que le module n'apparaît plus dans la page de configuration des modules d'un site client
- [ ] Le module sera soumis aux mêmes règles que les autres modules platform (BUG-027)
- [ ] Test de non-régression ajouté
- [ ] Tous les tests passent
- [ ] Build OK

## Statut : CORRIGÉ
