## Description
Quand l'utilisateur demarre un nouveau calibrage, tous les bacs de la vague sont grises (non-cliquables). Il est impossible de selectionner un bac source.

**Ce qui se passe :** Tous les bacs apparaissent dans la section "vide" avec opacity-50 et ne sont pas cliquables.
**Ce qui devrait se passer :** Les bacs contenant des poissons devraient etre selectionnables.

## Cause racine
Deux problemes lies :

### 1. Creation de vague : `nombrePoissons` jamais initialise
Dans `src/lib/queries/vagues.ts`, `createVague()` assigne les bacs avec seulement `vagueId` :
```ts
await tx.bac.updateMany({
  where: { id: { in: data.bacIds }, siteId },
  data: { vagueId: vague.id }, // nombrePoissons reste null !
});
```
`nombrePoissons` reste `null` sur tous les bacs.

### 2. Calibrage : bacs avec nombrePoissons=null/0 grises
Dans `src/components/calibrage/step-sources.tsx` :
```ts
const bacsAvecPoissons = bacs.filter((b) => (b.nombrePoissons ?? 0) > 0);
const bacsVides = bacs.filter((b) => (b.nombrePoissons ?? 0) === 0);
```
Tous les bacs tombent dans `bacsVides` â†’ grises et non-cliquables.

## Solution (ADR-023)
Voir `docs/decisions/ADR-023-bac-stocking-distribution.md`

**ResumĂ© :**
1. **Backend** : Modifier `CreateVagueDTO` â€” remplacer `bacIds: string[]` par `bacDistribution: { bacId: string, nombrePoissons: number }[]`. Chaque bac recoit `nombrePoissons`, `nombreInitial`, `poidsMoyenInitial` a la creation.
2. **Frontend** : Ajouter des inputs de distribution inline dans le formulaire de creation de vague. Bouton "Repartir equitablement" en helper. Indicateur de total en temps reel.
3. **Backward compat** : Dans `step-sources.tsx`, les bacs avec `nombrePoissons=null` mais assignes a la vague doivent etre selectionnables (afficher "quantite inconnue" au lieu de les griser).
4. **Validation** : `sum(distribution) === nombreInitial` obligatoire.
