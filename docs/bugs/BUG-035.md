# BUG-035 — Analytics aliments : filtre bloque l'UI sans feedback
**Sévérité :** Haute
**Détecté par :** @user (Ronald)
**Sprint :** UX
**Fichier(s) :** src/components/analytics/feed-filters.tsx, src/app/(farm)/analytics/aliments/

## Description
Quand l'utilisateur sélectionne un filtre dans la page analytics aliments, le Select ne reflète pas le choix immédiatement. L'UI reste figée sans aucun indicateur de chargement jusqu'à ce que le serveur réponde. Très mauvaise expérience utilisateur.

## Étapes de reproduction
1. Aller sur /analytics/aliments
2. Ouvrir le Select "Phase"
3. Sélectionner une phase (ex: "Grossissement")
4. Observer : le Select ne change pas visuellement, rien n'indique qu'une action est en cours
5. Après 1-3 secondes, la page entière se recharge avec le nouveau filtre

## Cause racine
Triple problème :
1. `feed-filters.tsx` lit les valeurs directement depuis `searchParams` (pas d'état local)
2. `router.push()` déclenche un re-render serveur complet — le Select revient à l'ancienne valeur pendant le fetch
3. Pas de `loading.tsx` dans `src/app/(farm)/analytics/aliments/` — zéro feedback visuel

## Fix
- Ajouter un état local pour chaque filtre (mise à jour immédiate de l'UI)
- Utiliser `useTransition` pour navigation non-bloquante avec `isPending`
- Ajouter `loading.tsx` skeleton dans `src/app/(farm)/analytics/aliments/`
- Afficher indicateur de chargement dans la zone résultats pendant `isPending`
- Voir plan détaillé : `docs/decisions/PLAN-ux-filters-loading.md` §P2
