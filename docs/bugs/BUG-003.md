# BUG-003 — Hydration mismatch sur body (extension navigateur)
**Sévérité :** Basse
**Détecté par :** @db-specialist
**Sprint :** 6
**Fichier(s) :** src/app/layout.tsx

## Description
En dev (Next.js 16.1.6 Turbopack), une erreur console de type hydration mismatch apparaît sur la page `/login`. Le HTML rendu côté serveur ne contient pas l'attribut `data-atm-ext-installed="1.30.01"`, mais le DOM client l'a car une extension navigateur l'injecte sur le `<body>`.

**Ce qui se passe :** Erreur console "Hydration failed because the server rendered HTML didn't match the client."
**Ce qui devrait se passer :** Aucune erreur console.

## Étapes de reproduction
1. Installer une extension navigateur qui injecte des attributs sur `<body>` (ex: gestionnaire de mots de passe)
2. Lancer `npm run dev`
3. Ouvrir `/login` dans le navigateur
4. Observer l'erreur hydration mismatch dans la console

## Cause racine
Les extensions navigateur injectent des attributs custom (ex: `data-atm-ext-installed`) sur le `<body>` après le rendu SSR. Next.js détecte un écart entre le HTML SSR et le DOM client et lève une erreur.

Ce n'est pas un bug de code — c'est un problème d'environnement navigateur. Le fix standard Next.js est d'ajouter `suppressHydrationWarning` sur `<body>`.
