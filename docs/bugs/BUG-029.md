# BUG-029 — Platform entity siteId not enforced to platform site
**Sévérité :** Moyenne
**Détecté par :** @project-manager
**Sprint :** 34-35 (Commissions, Remises)
**Fichier(s) :** src/app/api/commissions/, src/app/api/remises/, src/lib/queries/

## Description
Les modèles platform-only (`CommissionIngenieur`, `PortefeuilleIngenieur`, `Remise` globale)
ont un champ `siteId` (R8) mais rien ne garantit que ce `siteId` pointe vers le site plateforme.

Actuellement, le `siteId` est rempli avec le `activeSiteId` de la session, qui peut être un site
client quelconque. Cela crée des données incohérentes : une commission d'ingénieur pourrait être
rattachée à un site client au lieu du site plateforme.

## Étapes de reproduction
1. Se connecter en tant qu'admin
2. Sélectionner un site client comme `activeSiteId`
3. Créer une commission ou une remise globale → le `siteId` sera celui du site client

## Cause racine
Les API routes utilisent `session.activeSiteId` pour remplir `siteId` sans vérifier que
l'entité est de type plateforme et doit pointer vers le site plateforme.

## Fix
- [ ] Identifier toutes les routes API qui créent des entités platform : commissions, portefeuille, remises globales
- [ ] Pour ces routes, forcer `siteId` = ID du site plateforme (via `getPlatformSite()` de BUG-025) au lieu de `session.activeSiteId`
- [ ] Ajouter validation : ces routes ne doivent être accessibles que depuis le contexte plateforme (`activeSiteId` = site platform)
- [ ] Dépend de BUG-025 (isPlatform flag + helper getPlatformSite)
- [ ] Mettre à jour les données seed si des entités platform ont le mauvais siteId
- [ ] Test de non-régression ajouté
- [ ] Tous les tests passent
- [ ] Build OK
