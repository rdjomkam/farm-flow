# BUG-047 — Safari iOS (non-PWA) : bandes safe-area haut et bas transparentes
**Sévérité :** Haute
**Détecté par :** Ronald (test browser Safari iOS)
**Sprint :** 12
**Fichier(s) :** src/app/globals.css

## Description
Sur Safari iOS en mode navigateur (non-PWA), deux problèmes de bandes transparentes :

1. **Bas** : la bande `safe-area-inset-bottom` clignote transparent lors des transitions de la URL bar Safari. Le `html::after` positionné à `bottom: 0` avec `height: env(safe-area-inset-bottom)` ne couvre pas les frames où Safari fait fluctuer transitoirement la valeur vers 0 pendant l'animation de la barre.

2. **Haut** : aucun backdrop dédié n'existe pour la bande `safe-area-inset-top`. Le FarmHeader couvre son propre fond via `position: sticky`, mais (a) il peut se "décoller" lors des transitions de page, (b) les pages sans FarmHeader (login, select-site, pages d'erreur) n'ont aucune protection.

Extension directe de BUG-043 qui couvrait uniquement la bande du bas en mode PWA.

## Étapes de reproduction
1. Ouvrir l'application sur Safari iOS (non-PWA, dans le navigateur)
2. Naviguer entre des pages (ex. / → /vagues → /)
3. Observer la bande en bas lors de l'animation de la URL bar : transparent pendant quelques frames
4. Ouvrir /login ou /settings/sites : pas de FarmHeader, la bande en haut est transparente

## Cause racine
- **Bas** : `html::after { bottom: 0; height: env(safe-area-inset-bottom) }` — quand Safari anime la URL bar, `env(safe-area-inset-bottom)` fluctue transitoirement vers 0, rendant le pseudo-élément de hauteur nulle pendant cette frame.
- **Haut** : aucun `html::before` n'existait pour couvrir `safe-area-inset-top` indépendamment du FarmHeader.

## Fix
- [x] `html::before` ajouté : backdrop fixe pour `safe-area-inset-top`, z-index 49 (sous le header z-50)
- [x] `html::after` étendu : `bottom: -60px; height: calc(env(safe-area-inset-bottom) + 60px)` pour absorber les frames où Safari repousse transitoirement l'inset vers 0 pendant les animations de URL bar
- [x] Test de non-régression ajouté dans `src/components/layout/__tests__/bottom-nav.test.tsx`
- [x] Tous les tests passent (hors 37 échecs pré-existants inchangés)
- [x] Build OK

## Relation avec BUG-043
BUG-043 v2 avait introduit le `html::after` pour la bande du bas en mode PWA. BUG-047 étend cette protection :
- Absorbe les glitches de la URL bar Safari non-PWA (extension du `html::after`)
- Ajoute la protection manquante pour la bande du haut (`html::before`)
