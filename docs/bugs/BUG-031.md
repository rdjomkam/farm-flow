# BUG-031 — GlobalLoadingBar animation invisible (keyframe manquant)
**Severite :** Haute
**Detecte par :** @architect
**Sprint :** N/A (transversal)
**Fichier(s) :** src/components/ui/global-loading-bar.tsx, src/app/globals.css

## Description
Le `GlobalLoadingBar` utilise la classe CSS `animate-loading-bar` (ligne 41 de global-loading-bar.tsx) mais le `@keyframes loading-bar` correspondant n'est jamais defini dans `globals.css`. Le resultat : la barre de chargement est rendue (track visible) mais l'indicateur ne s'anime pas — il reste invisible ou statique. Meme les routes qui ont deja un `loading.tsx` (bacs, vagues, stock, finances, etc.) n'affichent aucun feedback visuel de chargement.

## Etapes de reproduction
1. Naviguer vers /bacs depuis le dashboard
2. Observer le haut de l'ecran — aucune animation de barre de progression
3. Inspecter l'element `.animate-loading-bar` dans DevTools — la propriete `animation` est invalide

## Cause racine
La classe `animate-loading-bar` est utilisee dans le composant mais le `@keyframes` correspondant n'a jamais ete ajoute a `globals.css`.

## Fix
- [x] Ajouter `@keyframes loading-bar` et `.animate-loading-bar` dans `src/app/globals.css`
- [x] Verifier visuellement que la barre s'anime correctement
- [x] Build OK
