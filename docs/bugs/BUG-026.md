# BUG-026 — Platform permissions assignable to non-platform sites
**Sévérité :** Haute
**Détecté par :** @project-manager
**Sprint :** 7 (Multi-tenancy)
**Fichier(s) :** src/lib/permissions-constants.ts, src/lib/auth/permissions-server.ts, src/app/api/sites/[id]/roles/

## Description
Les permissions platform-only (PLANS_GERER, ABONNEMENTS_GERER, COMMISSIONS_GERER, REMISES_GERER,
PORTEFEUILLE_GERER, COMMISSION_PREMIUM, etc.) peuvent être assignées à des SiteRoles sur des sites
clients normaux. Rien n'empêche un admin de ferme d'attribuer PLANS_GERER à un rôle sur son site.

De plus, quand `getServerPermissions()` charge les permissions d'un utilisateur, il retourne toutes
les permissions du SiteRole sans filtrer les permissions platform-only. Un utilisateur sur un site
non-platform ne devrait jamais avoir de permissions platform dans son contexte.

## Étapes de reproduction
1. Se connecter en tant qu'admin d'un site client
2. Créer un SiteRole avec `PLANS_GERER` dans les permissions
3. Résultat : le rôle est créé sans erreur → la permission platform est assignable partout

## Cause racine
Le système de permissions ne distingue pas les permissions platform des permissions site.
Toutes les 59 permissions sont dans le même enum sans métadonnée de niveau.

## Fix
- [ ] Ajouter un flag/classification `isPlatform` sur chaque permission dans `src/lib/permissions-constants.ts` (ex: `PLATFORM_PERMISSIONS: Permission[]`)
- [ ] Définir la liste des permissions platform-only : `PLANS_GERER`, `ABONNEMENTS_VOIR`, `ABONNEMENTS_GERER`, `REMISES_GERER`, `COMMISSIONS_VOIR`, `COMMISSIONS_GERER`, `COMMISSION_PREMIUM`, `PORTEFEUILLE_VOIR`, `PORTEFEUILLE_GERER`
- [ ] Valider côté API (création/modification SiteRole) : rejeter les permissions platform si le site n'est pas `isPlatform`
- [ ] Modifier `getServerPermissions()` : ne retourner les permissions platform que si `activeSiteId` pointe vers le site platform
- [ ] Modifier l'UI de gestion des rôles : ne pas afficher les permissions platform pour les sites non-platform
- [ ] Mettre à jour les rôles système (`SYSTEM_ROLES`) : les permissions platform ne doivent être incluses que dans les rôles du site platform
- [ ] Test de non-régression ajouté
- [ ] Tous les tests passent
- [ ] Build OK
