# BUG-038 — Dépenses : onglet ne reset pas au changement de catégorie
**Sévérité :** Moyenne
**Détecté par :** @architect
**Sprint :** UX
**Fichier(s) :** src/components/depenses/depenses-list-client.tsx

## Description
La page dépenses a deux axes de filtrage : un Select (catégorie) et des Tabs (statut paiement). Les Tabs utilisent `defaultValue` (non-contrôlé). Quand la catégorie change, l'onglet actif ne se reset pas — il peut afficher un onglet vide si la nouvelle catégorie n'a pas de données dans ce statut.

## Étapes de reproduction
1. Aller sur /depenses
2. Sélectionner un onglet spécifique (ex: "Payées")
3. Changer la catégorie dans le Select
4. Observer : l'onglet "Payées" reste actif même si la nouvelle catégorie n'a aucune dépense payée

## Cause racine
`<Tabs defaultValue="toutes">` est non-contrôlé (ligne 177). React ne reset pas le state interne de Radix quand les props externes changent.

## Fix
- Passer en mode contrôlé : `<Tabs value={tab} onValueChange={setTab}>`
- Ajouter `handleCategorieChange` qui reset `tab` à "toutes"
- Voir plan détaillé : `docs/decisions/PLAN-ux-filters-loading.md` §P7
