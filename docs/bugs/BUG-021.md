# BUG-021 — Modules Sprints 33-35 absents du menu mobile (hamburger)
**Severite :** Haute
**Detecte par :** @project-manager
**Sprint :** 38
**Fichier(s) :** src/components/layout/hamburger-menu.tsx

## Description
Les modules ajoutes dans les Sprints 33-35 sont presents dans la sidebar desktop (`sidebar.tsx`) mais absents du menu hamburger mobile (`hamburger-menu.tsx`). Sur mobile (360px), ces pages sont inaccessibles depuis la navigation.

**Modules manquants dans le hamburger menu :**
1. **Abonnement** — `/mon-abonnement`, `/tarifs` (Sprint 33, gate: `ABONNEMENTS_VOIR`)
2. **Admin Abonnements** — `/admin/abonnements` (Sprint 33, gate: `ABONNEMENTS_GERER`)
3. **Portefeuille** — `/mon-portefeuille` (Sprint 34, gate: `PORTEFEUILLE_VOIR`)
4. **Admin Commissions** — `/admin/commissions` (Sprint 34, gate: `COMMISSIONS_GERER`)
5. **Admin Remises** — `/admin/remises` (Sprint 35, gate: `REMISES_GERER`)

**Ce qui fonctionne :** La sidebar desktop affiche correctement tous ces modules avec les permission gates.

## Etapes de reproduction
1. Se connecter en tant qu'admin sur mobile (< 768px)
2. Taper sur le bouton "Plus" dans la barre de navigation du bas
3. Observer que le menu hamburger ne contient pas les liens vers Abonnement, Portefeuille, Admin Abonnements, Admin Commissions, Admin Remises
4. Comparer avec la sidebar desktop qui affiche bien ces modules

## Cause racine
Le composant `hamburger-menu.tsx` duplique la liste `modulesAdminGerant` de `sidebar.tsx` mais n'a pas ete mis a jour lors des Sprints 33-35 pour ajouter les nouveaux modules. De meme, le dictionnaire `PHASE3_MODULE_PERMISSIONS` du hamburger menu ne contient pas les permissions gates des nouveaux modules (seuls `ACTIVER_PACKS`, `MONITORING_CLIENTS`, `UTILISATEURS_VOIR` sont presents).

## Fix
- [x] Ajouter les 5 modules manquants dans `modulesAdminGerant` de `hamburger-menu.tsx` (copier depuis `sidebar.tsx`)
- [x] Ajouter les 5 permission gates manquantes dans `PHASE3_MODULE_PERMISSIONS` de `hamburger-menu.tsx`
- [x] Importer les icones manquantes (`CreditCard`, `ShieldCheck`) dans `hamburger-menu.tsx`
- [x] Test de non-regression ajoute (45 tests dans hamburger-menu.test.ts)
- [x] Tous les tests passent
- [x] Build OK
