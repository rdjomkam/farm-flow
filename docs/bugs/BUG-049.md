# BUG-049 — Guard : `isEntrant` par bac au lieu de par relevé (transferts intra-vague miscomptés)

**Sévérité :** Haute
**Détecté par :** @architect (analyse en session)
**Sprint :** GD (Guard Discrimination)
**Fichier(s) :** `src/lib/guards/assignation-invariant.ts`

## Description

Le guard `verifyAssignationInvariant` calcule `isEntrant` **par bac** (via `TransfertGroupe.bacDestId`), pas par relevé. Conséquence : si un bac est à la fois source d'un TRANSFERT (sortant) et destination d'un autre TRANSFERT (entrant), **tous ses relevés TRANSFERT** sont comptés selon le signe du dernier trouvé → replay faux.

## Étapes de reproduction

1. Vague `cmplrrba6000101qwazzjca26` (Vague-26-03-Prep), Bac 11 :
   - COMPTAGE 09/06 = 2000
   - TRANSFERT sortant 10/06 vers Bac 05 (vague différente) = 1776
   - AssignationBac actuelle = 936 (post-arrivée informelle de Bac 12)
2. Créer un TransfertGroupe intra-vague : Bac 12 → Bac 11, 712 poissons
3. Guard s'exécute : `entrantBacIds` contient Bac 11 (destination du nouveau TG)
4. Replay : `2000 + 1776 (sortant miscompté +) + 712 = 4488` ≠ 936 actuel → `ConservationError`

## Cause racine

`src/lib/guards/assignation-invariant.ts:114-125` charge `TransfertGroupe.bacDestId` et bâtit un `Set<bacId>` (`entrantBacIds`). Ligne 183-186 :

```ts
} else if (r.typeReleve === TypeReleve.TRANSFERT) {
  expected += isEntrant
    ? (r.nombreTransferes ?? 0)
    : -(r.nombreTransferes ?? 0);
}
```

`isEntrant` s'applique à **tous** les TRANSFERT relevés du bac, mais un même bac peut avoir des TRANSFERT sortants ET entrants.

## Fix

- [ ] Refactor : discrimination `isEntrant` **par relevé** via `Releve.transfertGroupeId` + comparaison au `TransfertGroupe.bacSourceId` (sortant) vs `bacDestId` (entrant)
- [ ] Charger les TransfertGroupe des relevés impactés (bulk query par `transfertGroupeId`)
- [ ] Tests unitaires nouveaux :
  1. Bac source dans TG intra-vague → sortant (-)
  2. Bac dest dans TG intra-vague → entrant (+)
  3. Bac source + dest de 2 TG différents dans même vague → discrimination correcte par relevé
- [ ] Data-fix rétroactif SQL prod pour Vague-26-03-Prep (Bac 08, 12, 11 anti-pattern « COMPTAGES déguisés »)
