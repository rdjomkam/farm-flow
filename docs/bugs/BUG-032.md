# BUG-032 — Aucun feedback de navigation sur ~45 routes (loading.tsx manquants)
**Severite :** Haute
**Detecte par :** @architect
**Sprint :** N/A (transversal)
**Fichier(s) :** ~45 segments de route dans src/app/

## Description
Quand l'utilisateur clique sur un lien de navigation (ex: "Analytics par bacs", "Analytics par vague", pages alevins, ventes, clients, depenses, etc.), rien ne se passe visuellement pendant 2-4 secondes jusqu'a ce que la page charge. L'utilisateur pense que l'app est buguee. Le probleme : ces segments de route n'ont pas de fichier `loading.tsx`, donc Next.js ne declenche pas de boundary Suspense et le `NavigationLoader`/`GlobalLoadingBar` ne s'active jamais.

Le root `src/app/loading.tsx` existe mais ne couvre PAS les sous-segments imbriques (ex: `/analytics/bacs` ne beneficie pas du root loading).

## Etapes de reproduction
1. Ouvrir l'app sur mobile (360px)
2. Naviguer vers le dashboard
3. Cliquer sur "Analytics par bacs" ou "Analytics par vague"
4. Observer : rien ne se passe pendant 2-4 secondes, puis la page apparait d'un coup

## Cause racine
Fichiers `loading.tsx` manquants dans les segments de route concernes. Sans boundary Suspense, Next.js ne montre aucun etat de chargement pendant la navigation SSR.

## Fix
- [ ] Creer ~43 fichiers `loading.tsx` identiques (pattern `NavigationLoader` existant)
- [ ] Verifier que la navigation montre le feedback sur toutes les routes
- [ ] Build OK
