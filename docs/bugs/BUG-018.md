# BUG-018 ‚Äî prisma.notification undefined au runtime (TypeError findMany)

**Severite :** Critique
**Detecte par :** @ronald
**Sprint :** 11
**Fichier(s) :** src/lib/queries/alertes.ts:127, src/app/notifications/page.tsx:13

## Description

La page `/notifications` crash au chargement avec un `TypeError: Cannot read properties of undefined (reading 'findMany')`. Le probleme est que `prisma.notification` est `undefined` au runtime. Meme symptome que BUG-017 (prisma.reproducteur undefined) ‚Äî meme cause racine (client Prisma non regenere).

## Etapes de reproduction

1. Lancer l'app en dev (`npm run dev`)
2. Naviguer vers `/notifications`
3. La page crash avec le TypeError

## Cause racine

**Identique a BUG-017 : client Prisma non regenere apres la migration Sprint 11 (`20260311100000_add_alertes_planning`).**

Le modele `Notification` a ete ajoute dans la migration `add_alertes_planning` avec les tables `ConfigAlerte`, `Notification`, `Activite`. Le client genere `src/generated/prisma/` n'avait pas ete regenere apres cette migration, donc la propriete `prisma.notification` n'existait pas sur le singleton `PrismaClient` en memoire.

**Verification effectuee :**
- Table `Notification` presente en DB : OUI (confirmee via `\dt`)
- Migration `20260311100000_add_alertes_planning` appliquee : OUI (finished_at renseigne)
- Fichier `src/generated/prisma/models/Notification.ts` existait : OUI (mais client pas regenere)
- Getter `get notification()` dans class.ts apres `prisma generate` : ligne 419

## Fix applique

Meme fix que BUG-017 ‚Äî les trois bugs (017, 018, 019) partagent la meme cause racine et ont ete corriges ensemble :

### 1. Regeneration du client
```bash
npx prisma generate
```

Le getter expose dans `src/generated/prisma/internal/class.ts` (ligne 419) :
```typescript
get notification(): Prisma.NotificationDelegate<ExtArgs, { omit: OmitOpts }>;
```

### 2. Nettoyage cache
```bash
rm -rf .next
```

### 3. Prevention dans `package.json`
```json
"build": "prisma generate && next build",
"postinstall": "prisma generate"
```
