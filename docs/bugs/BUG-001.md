# BUG-001 — Login par telephone manquant — email doit devenir optionnel
**Severite :** Haute
**Detecte par :** @project-manager
**Sprint :** 6
**Fichier(s) :** prisma/schema.prisma, src/types/models.ts, docs/decisions/005-authentification.md, docs/TASKS.md (stories 6.2, 6.4, 6.6, 6.7, 6.9)

## Description
Les pisciculteurs au Cameroun n'ont pas tous une adresse email. Le telephone doit etre une alternative pour se connecter.

Actuellement le modele User n'a que `email` comme identifiant unique. Il faut :

- Ajouter `phone String? @unique` sur le modele User
- Rendre `email` nullable (`String? @unique`) au lieu de `String @unique`
- Au moins un des deux (email ou phone) doit etre requis (validation applicative, pas DB)
- Le login doit accepter `{ identifier, password }` ou `identifier` peut etre un email ou un numero de telephone
- Le register doit permettre de s'inscrire avec email et/ou phone
- Format telephone : camerounais (+237 6XX XX XX XX ou +237 2XX XX XX XX)

## Etapes de reproduction
1. Ouvrir la page de login (`/login`)
2. Constater qu'il n'y a qu'un champ email — pas de champ telephone
3. Un pisciculteur sans email ne peut pas se connecter

## Cause racine
L'ADR 005-authentification et le schema User ont ete concus avec email uniquement. Le cas d'usage terrain (pisciculteurs sans email) n'a pas ete pris en compte.

## Fix
- [ ] Migration Prisma : `email` nullable, ajout `phone String? @unique`
- [ ] Mise a jour `src/types/models.ts` : interface User avec `email?` et `phone?`
- [ ] Mise a jour ADR 005 : ajouter le login par telephone
- [ ] API `/api/auth/login` : accepter `identifier` (email ou phone) + `password`
- [ ] API `/api/auth/register` : accepter email et/ou phone
- [ ] Validation applicative : au moins un des deux requis
- [ ] Validation format telephone camerounais (+237...)
- [ ] UI login : champ identifier avec placeholder "Email ou telephone"
- [ ] UI register : champs email (optionnel) et telephone (optionnel), au moins un requis
- [ ] Tests unitaires et d'integration
- [ ] Tous les tests passent
- [ ] Build OK
