# BUG-002 — Le prefixe +237 ne doit jamais etre saisi par l'utilisateur
**Severite :** Moyenne
**Detecte par :** @project-manager
**Sprint :** 6
**Fichier(s) :** src/app/api/auth/login/route.ts, src/app/api/auth/register/route.ts, src/lib/auth/, src/app/login/page.tsx, src/app/register/page.tsx

## Description
L'utilisateur ne doit jamais saisir le code pays (+237) dans le champ telephone. Il entre uniquement le numero local (ex: 6 99 99 99 99). Le backend se charge d'ajouter le prefixe +237 avant stockage et de le normaliser lors du login.

Regles :
- UI : champ telephone sans prefixe, placeholder "6XX XX XX XX"
- Backend register : ajouter automatiquement +237 avant stockage en DB
- Backend login : si l'identifier ressemble a un numero local (commence par 6 ou 2), prefixer +237 avant la recherche en DB
- Validation backend : verifier que le numero local fait 9 chiffres (format camerounais)
- Ne jamais exposer le +237 a l'utilisateur dans l'UI

## Etapes de reproduction
1. Ouvrir la page register ou login
2. L'utilisateur ne sait pas s'il doit taper +237 ou pas
3. Risque de doublons en DB (avec et sans prefixe)

## Cause racine
BUG-001 prevoyait un format +237 cote utilisateur. Le cas d'usage terrain montre que les pisciculteurs tapent uniquement le numero local.
