# BUG-024 — PlansGrid utilise des prix hardcodés au lieu des prix DB
**Sévérité :** Haute
**Détecté par :** @project-manager
**Sprint :** 33
**Fichier(s) :** src/components/abonnements/plans-grid.tsx

## Description
Même si BUG-023 est corrigé et que les plans arrivent depuis la DB, le composant `PlansGrid`
ignore les prix du plan (`plan.prixMensuel`, `plan.prixTrimestriel`, `plan.prixAnnuel`) et utilise
à la place les constantes hardcodées `PLAN_TARIFS` et `PLAN_LIMITES` de `src/lib/abonnements-constants.ts`.

Lignes concernées dans `plans-grid.tsx` :
- **Ligne 146** : `const tarif = PLAN_TARIFS[plan.typePlan]?.[periode]` → devrait être `plan.prixMensuel` / `plan.prixTrimestriel` / `plan.prixAnnuel`
- **Ligne 147** : `const tarifMensuel = PLAN_TARIFS[plan.typePlan]?.[PeriodeFacturation.MENSUEL]` → idem
- **Ligne 93-94** (dans `getEconomie`) : même problème avec `PLAN_TARIFS`
- **Ligne 149** : `const limites = PLAN_LIMITES[plan.typePlan]` → devrait utiliser `plan.limitesBacs`, `plan.limitesVagues`, `plan.limitesSites`
- **Lignes 112-116** : `periodesDisponibles` filtre aussi via `PLAN_TARIFS` au lieu des champs du plan

Cela rend inutile le stockage des prix en DB et empêche un admin de modifier les tarifs via l'UI CRUD (Sprint 38).

## Étapes de reproduction
1. Modifier le prix d'un plan en DB (ex: changer prixMensuel de ELEVEUR)
2. Ouvrir `/tarifs`
3. Observer : l'ancien prix hardcodé est affiché, pas le prix DB

## Cause racine
Le composant a été codé avec des constantes en dur avant que l'API ne soit prête (Sprint 33),
et n'a jamais été mis à jour pour utiliser les données réelles du plan.

## Fix proposé
Remplacer toutes les références à `PLAN_TARIFS` et `PLAN_LIMITES` par les champs du plan :

```typescript
// Prix selon la période sélectionnée
function getTarifForPeriode(plan: PlanAbonnement, periode: PeriodeFacturation): number | null {
  switch (periode) {
    case PeriodeFacturation.MENSUEL: return plan.prixMensuel;
    case PeriodeFacturation.TRIMESTRIEL: return plan.prixTrimestriel;
    case PeriodeFacturation.ANNUEL: return plan.prixAnnuel;
  }
}

// Limites depuis le plan
const limites = {
  limitesBacs: plan.limitesBacs,
  limitesVagues: plan.limitesVagues,
  limitesSites: plan.limitesSites,
};
```

Note : les `Decimal` Prisma doivent être convertis en `number` côté serveur avant d'être passés au Client Component (via `Number(plan.prixMensuel)` ou `.toNumber()`).
