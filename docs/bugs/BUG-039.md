# BUG-039 — Planning : mois hors fenêtre préchargée affichent vide
**Sévérité :** Moyenne
**Détecté par :** @architect
**Sprint :** UX
**Fichier(s) :** src/components/planning/planning-client.tsx

## Description
Le calendrier du planning ne charge des données que pour le mois initial. Quand l'utilisateur navigue vers un mois futur ou passé, les jours du calendrier sont vides — aucun fetch n'est déclenché pour le nouveau mois.

## Étapes de reproduction
1. Aller sur /planning
2. Observer les activités du mois courant (correctes)
3. Naviguer au mois suivant avec la flèche
4. Observer : calendrier vide, aucune activité affichée

## Cause racine
Les données sont préchargées côté serveur pour le mois courant uniquement. Le composant client filtre en mémoire — pas de fetch pour les mois hors fenêtre.

## Fix
- Ajouter un fetch client-side quand le mois visible change
- Utiliser le service existant pour charger les activités du nouveau mois
- Voir plan détaillé : `docs/decisions/PLAN-ux-filters-loading.md` §P5
