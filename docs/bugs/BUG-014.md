# BUG-014 — 500 lors du changement de site actif (PUT /api/auth/site)

**Severite :** Critique
**Detecte par :** @ronald
**Assigne a :** @tester (verification uniquement — fix deja applique)
**Sprint :** 9 (CR-009)
**Fichier(s) :** src/lib/queries/sites.ts, src/app/api/auth/site/route.ts

## Description

Quand un utilisateur selectionne un site (PUT /api/auth/site avec `{ siteId: "..." }`), le serveur retourne une erreur 500 : "Erreur serveur lors du changement de site."

L'erreur sous-jacente est :
```
PrismaClientValidationError: Unknown field 'siteRole' for include statement on model 'SiteMember'.
Available options are marked with ?.
```

**Impact :** TOUTES les fonctionnalites qui utilisent `getSiteMember()` sont cassees (changement de site, verification de permissions des membres, routes API protegees par `requirePermission()`). L'application est inutilisable apres login.

## Etapes de reproduction

1. Se connecter a l'application
2. Cliquer sur un site pour le selectionner
3. Observer l'erreur 500

**Reproduction curl :**
```bash
curl -X PUT http://localhost:3000/api/auth/site \
  -H "Content-Type: application/json" \
  -H "Cookie: farm-flow-session=<session_token>" \
  -d '{"siteId":"<site_id>"}'
# Retourne: {"status":500,"message":"Erreur serveur lors du changement de site."}
```

## Cause racine

Le client Prisma genere (`src/generated/prisma/`) est desynchronise avec le schema `prisma/schema.prisma` apres les modifications de CR-009.

CR-009 a ajoute :
- Le modele `SiteRole` dans schema.prisma
- La relation `siteRole SiteRole @relation(...)` sur `SiteMember`
- Le champ `siteRoleId` sur `SiteMember`

La fonction `getSiteMember()` dans `src/lib/queries/sites.ts` fait :
```typescript
include: { siteRole: true }
```

Mais le client Prisma genere n'avait pas ete regenere apres la migration, donc il ne connaissait pas le champ `siteRole` sur le modele `SiteMember`.

## Fix

1. Regenerer le client Prisma :
```bash
npx prisma generate
```

2. Supprimer le cache Next.js :
```bash
rm -rf .next
```

3. Redemarrer le serveur de developpement :
```bash
npm run dev
```

4. Verifier que PUT /api/auth/site retourne 200 :
```bash
curl -X PUT http://localhost:3000/api/auth/site \
  -H "Content-Type: application/json" \
  -H "Cookie: farm-flow-session=<session_token>" \
  -d '{"siteId":"<site_id>"}'
# Doit retourner: {"success":true,"activeSiteId":"...","siteRole":{...}}
```
