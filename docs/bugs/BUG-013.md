# BUG-013 — Conteneur permissions non colle au bas du dialog mobile

**Severite :** Moyenne
**Detecte par :** @code-reviewer
**Assigne a :** @developer
**Sprint :** 7
**Fichier(s) :** src/components/sites/member-actions-dialog.tsx

## Description
Dans la vue permissions du dialog membre, la liste des permissions et les boutons d'action (Defaut, Annuler, Enregistrer) flottent au milieu de l'ecran sur mobile au lieu d'occuper tout l'espace disponible. Le dialog est plein ecran sur mobile (`inset-0`, `flex h-full flex-col`) mais ni le conteneur scrollable (ligne 344, `max-h-[60vh]`) ni la rangee de boutons (ligne 381) ne sont pousses vers le bas du flex.

## Etapes de reproduction
1. Ouvrir l'app sur mobile (360px)
2. Aller dans Sites > Membres
3. Cliquer sur un membre > "Modifier permissions"
4. Observer que la liste des permissions et les boutons ne remplissent pas la hauteur de l'ecran
5. Il y a ~40vh d'espace vide sous les boutons

## Cause racine
Le conteneur scrollable des permissions (ligne 344) a `max-h-[60vh]` mais pas `flex-1` pour occuper l'espace restant. La rangee de boutons (ligne 381) n'a pas `mt-auto`. Les deux elements sont des enfants directs du flex column du dialog mais aucun ne s'etire pour remplir la hauteur disponible.

## Fix propose
- `src/components/sites/member-actions-dialog.tsx` ligne 344 : ajouter `flex-1` au conteneur scrollable (remplacer `max-h-[60vh]` par `flex-1 min-h-0 overflow-y-auto`) pour qu'il prenne tout l'espace vertical disponible
- Ligne 381 : ajouter `mt-auto` a la rangee de boutons pour la coller en bas
