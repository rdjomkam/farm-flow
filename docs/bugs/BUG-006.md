# BUG-006 — Toast transparent, texte visible derriere
**Severite :** Basse
**Detecte par :** @team-lead (signale par utilisateur)
**Sprint :** 8
**Fichier(s) :** src/components/ui/toast.tsx

## Description
Les toasts success/error/info utilisent des fonds semi-transparents (`bg-success/10`, `bg-danger/10`, `bg-primary/10` = 10% opacite), ce qui rend le texte de la page visible derriere le toast. Le toast devrait avoir un fond opaque pour etre lisible.

Ce qui se passe : le texte de la page est visible a travers le fond du toast.
Ce qui devrait se passer : le toast a un fond opaque, le texte est parfaitement lisible.

## Etapes de reproduction
1. Effectuer une action qui declenche un toast (ex: creer un fournisseur, sauvegarder un produit)
2. Observer le toast qui apparait
3. Le texte de la page en dessous est visible a travers le fond du toast

## Cause racine
Les variantes success, error et info du composant toast utilisent des classes de fond a 10% d'opacite :
- `bg-success/10` (success)
- `bg-danger/10` (error)
- `bg-primary/10` (info)

Au lieu d'un fond opaque comme `bg-card`.

## Fix suggere
Remplacer `bg-*/10` par `bg-card` dans les variantes du composant toast.
