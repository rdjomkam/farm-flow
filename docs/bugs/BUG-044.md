# BUG-044 — Chart "Evolution du poids moyen" affiche une moyenne pondérée incorrecte

**Sévérité :** Haute
**Détecté par :** @claude (analyse via psql sur la prod)
**Sprint :** 12
**Fichier(s) :**
- `src/lib/queries/vagues.ts` (getVagueById, getVagueByIdWithReleves)
- `src/components/pages/vague-detail-page.tsx` (appel computeVivantsByBac)

## Description

Le graphique "Evolution du poids moyen" sur la page de détail d'une vague affichait une
moyenne pondérée erronée lorsque la distribution de poissons entre bacs est inégale.

Pour la vague 26-02 (5500 poissons, bacs : 3500 / 1800 / 200) :
- Valeur affichée (buggy) : ~67.19 g
- Valeur correcte (getIndicateursVague) : ~47.45 g

## Étapes de reproduction

1. Créer une vague avec 3 bacs à distribution très inégale (ex. 3500 / 1800 / 200)
2. Ajouter des relevés de biométrie avec des poids moyens différents par bac
3. Ouvrir la page de détail de la vague → observer le graphique "Evolution du poids moyen"
4. Comparer avec la valeur calculée par getIndicateursVague → divergence significative

## Cause racine

`getVagueById` (et `getVagueByIdWithReleves`) construisent la liste finale de bacs en deux
étapes (union AssignationBac + Bac.vagueId). Le select sur `assignations.bac` ne sélectionnait
que `{ id, nom, volume }` — les champs `nombreInitial` et `poidsMoyenInitial` étaient
silencieusement supprimés.

La page `vague-detail-page.tsx` appelle ensuite `computeVivantsByBac(vague.bacs, ...)` qui
lit `bac.nombreInitial`. Avec `nombreInitial = undefined`, la condition
`bac.nombreInitial != null` est false → fallback sur la répartition uniforme
(floor(5500 / 3) = 1833 par bac au lieu de 3500 / 1800 / 200).

Résultat : bac-c (80 g) recevait 1833 vivants au lieu de 200, tirant la moyenne pondérée
vers le haut de façon artificielle.

Les autres call sites de `computeVivantsByBac` (gompertz API route, indicateurs.ts,
dashboard.ts) sélectionnaient `nombreInitial` directement sur le modèle Bac et n'étaient
pas affectés.

## Fix

- [x] `src/lib/queries/vagues.ts` — `getVagueById` : ajout de `nombreInitial` et
  `poidsMoyenInitial` dans le select `assignations.bac` et dans les deux maps UNION
- [x] `src/lib/queries/vagues.ts` — `getVagueByIdWithReleves` : même correctif
- [x] Test de non-régression ajouté dans `src/__tests__/survie-calculs.test.ts`
  (describe "BUG-044 — regression : weighted average chart avec distribution inégale", 3 tests)
- [x] Tous les tests passent (19/19 dans survie-calculs.test.ts, suite complète inchangée
  par rapport à la baseline)
- [x] Build OK (0 erreur TypeScript)
