# BUG-017 — prisma.reproducteur undefined au runtime (TypeError findMany)

**Severite :** Critique
**Detecte par :** @ronald
**Sprint :** 10
**Fichier(s) :** src/lib/queries/reproducteurs.ts:27, src/app/alevins/page.tsx:23

## Description

La page `/alevins` crash au chargement avec un `TypeError: Cannot read properties of undefined (reading 'findMany')`. Le probleme est que `prisma.reproducteur` est `undefined` au runtime alors que le modele `Reproducteur` existe dans le schema, la migration `20260310200000_add_alevins` est appliquee, et le client genere contient `src/generated/prisma/models/Reproducteur.ts`.

## Etapes de reproduction

1. Lancer l'app en dev (`npm run dev`)
2. Naviguer vers `/alevins`
3. La page crash avec le TypeError

## Cause racine

**Client Prisma non regenere apres les migrations Sprint 10 et Sprint 11.**

Avec Prisma 7 et le generator `prisma-client` avec output custom (`src/generated/prisma/`), le client n'est pas regenere automatiquement quand on ajoute de nouveaux modeles. Les migrations `20260310200000_add_alevins` et `20260311100000_add_alertes_planning` ont ete appliquees en base de donnees (tables creees), mais le client Prisma genere utilisait toujours une version anterieure sans les proprietes `reproducteur`, `ponte`, `lotAlevins`, `configAlerte`, `notification`, `activite`.

**Verification effectuee :**
- Migration appliquee : OUI — toutes les 13 migrations dans `_prisma_migrations` avec `finished_at` renseigne
- Tables en DB : OUI — `Reproducteur`, `Ponte`, `LotAlevins`, `Activite`, `Notification`, `ConfigAlerte` presentes dans PostgreSQL
- Client genere avant fix : manquait les getters pour les modeles Sprint 10/11
- Client genere apres `npx prisma generate` : expose bien `get reproducteur()`, `get ponte()`, `get lotAlevins()`, `get configAlerte()`, `get notification()`, `get activite()`

## Fix applique

### 1. Regeneration du client Prisma
```bash
npx prisma generate
```

Le client `src/generated/prisma/internal/class.ts` expose desormais les getters :
- Ligne 379 : `get reproducteur(): Prisma.ReproducteurDelegate<ExtArgs, ...>`
- Ligne 389 : `get ponte(): Prisma.PonteDelegate<ExtArgs, ...>`
- Ligne 399 : `get lotAlevins(): Prisma.LotAlevinsDelegate<ExtArgs, ...>`
- Ligne 409 : `get configAlerte(): Prisma.ConfigAlerteDelegate<ExtArgs, ...>`
- Ligne 419 : `get notification(): Prisma.NotificationDelegate<ExtArgs, ...>`
- Ligne 429 : `get activite(): Prisma.ActiviteDelegate<ExtArgs, ...>`

### 2. Nettoyage du cache Next.js
```bash
rm -rf .next
```

### 3. Prevention — scripts npm mis a jour dans `package.json`

Ajout de `prisma generate` dans le script `build` et d'un script `postinstall` pour garantir que le client est toujours regenere :

```json
"build": "prisma generate && next build",
"postinstall": "prisma generate",
"db:generate": "prisma generate",
"db:migrate": "prisma migrate deploy",
```

Cela garantit que :
- `npm install` regenere automatiquement le client
- `npm run build` regenere le client avant de compiler
- Le runtime Next.js verra toujours un client a jour
