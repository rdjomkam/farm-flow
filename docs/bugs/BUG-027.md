# BUG-027 — Platform modules visible on non-platform sites
**Sévérité :** Haute
**Détecté par :** @project-manager
**Sprint :** 38 (SiteModule platform/site)
**Fichier(s) :** src/lib/site-modules-config.ts, src/lib/auth/permissions-server.ts, src/components/layout/sidebar.tsx, src/components/layout/hamburger-menu.tsx

## Description
Les modules platform (ABONNEMENTS, COMMISSIONS, REMISES) sont hardcodés comme "toujours actifs"
dans `isModuleActive()` et dans `getServerSiteModules()`. Ils apparaissent dans la navigation
sidebar/hamburger même quand `activeSiteId` pointe vers un site client normal.

Un promoteur connecté à son site de ferme voit les liens Abonnements, Commissions, Remises dans
sa navigation (s'il a les permissions), alors que ces pages n'ont de sens que dans le contexte
du site plateforme.

## Étapes de reproduction
1. Se connecter en tant qu'admin avec permissions ABONNEMENTS_GERER
2. Sélectionner un site client (pas le site plateforme) comme `activeSiteId`
3. Observer la sidebar → les modules Abonnements, Commissions, Remises sont visibles

## Cause racine
`isModuleActive()` retourne toujours `true` pour les modules `level: "platform"`.
`getServerSiteModules()` ajoute systématiquement les modules platform au tableau retourné.
La navigation ne vérifie pas si le site actif est le site plateforme.

## Fix
- [ ] Modifier `getServerSiteModules()` dans `src/lib/auth/permissions-server.ts` : ne plus ajouter automatiquement les modules platform. Les ajouter UNIQUEMENT si `activeSiteId` pointe vers un site avec `isPlatform: true`
- [ ] Modifier `isModuleActive()` dans `src/lib/site-modules-config.ts` : accepter un paramètre `isPlatform` et ne retourner `true` pour les modules platform que si `isPlatform === true`
- [ ] Mettre à jour la sidebar et le hamburger menu pour passer le contexte `isPlatform` au filtrage
- [ ] Dépend de BUG-025 (isPlatform flag sur Site)
- [ ] Test de non-régression ajouté
- [ ] Tous les tests passent
- [ ] Build OK
