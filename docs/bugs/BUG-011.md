# BUG-011 — POST /api/releves avec consommations retourne 500
**Severite :** Haute
**Detecte par :** @project-manager (test manuel API)
**Assigne a :** @db-specialist (migration) + @developer (error logging)
**Sprint :** 8
**Fichier(s) :** src/app/api/releves/route.ts, src/lib/queries/releves.ts

## Description
Un POST sur `/api/releves` avec un tableau `consommations` retourne systematiquement `{"status":500,"message":"Erreur serveur lors de la creation du releve."}`. Les relevés sans consommations fonctionnent normalement.

Ce qui se passe : erreur 500 generique, le releve n'est pas cree.
Ce qui devrait se passer : le releve est cree avec les consommations, les mouvements de stock SORTIE sont generes, et le stock est decremente.

## Etapes de reproduction
1. S'authentifier (admin)
2. Envoyer un POST sur `/api/releves` avec le payload suivant :
```json
{
  "vagueId": "vague_01",
  "bacId": "bac_01",
  "typeReleve": "ALIMENTATION",
  "quantiteAliment": 50,
  "frequenceAliment": 13,
  "typeAliment": "COMMERCIAL",
  "consommations": [
    {"produitId": "prod_01", "quantite": 35},
    {"produitId": "prod_02", "quantite": 15}
  ]
}
```
3. Reponse : 500 au lieu de 201

## Cause racine
Hypothese principale : la migration `20260309150000_add_releve_consommation` n'a pas ete appliquee a la base de donnees PostgreSQL. Le fichier SQL existe et le client Prisma a ete regenere (le modele `ReleveConsommation` est present dans `src/generated/prisma/models/ReleveConsommation.ts`), mais la table `ReleveConsommation` et la colonne `MouvementStock.releveId` n'existent probablement pas dans la base.

Le code dans `queries/releves.ts:135` appelle `tx.releveConsommation.create()` et `queries/releves.ts:144` cree un `MouvementStock` avec `releveId` — les deux echouent si la migration n'est pas appliquee.

Probleme secondaire : le catch block dans `route.ts:308-330` avale l'erreur reelle sans la loguer, rendant le debug impossible.

- [x] Migration deja appliquee — table `ReleveConsommation` et colonne `MouvementStock.releveId` presentes
- [x] `console.error("[POST /api/releves] Error:", error)` deja present dans catch block ligne 309
- [x] Seed deja applique (produits avec stock disponible)
- [x] Test de non-regression ajoute : POST releve avec consommations → 201 (src/__tests__/api/releves.test.ts)
- [x] Test de non-regression ajoute : POST releve sans consommations → 201 (inchange)
