# BUG-020 — Select.Item avec value vide dans le formulaire releve
**Severite :** Haute
**Detecte par :** @user
**Sprint :** 13
**Fichier(s) :** src/components/releves/releve-form-client.tsx

## Description
Erreur React dans la console: `A <Select.Item /> must have a value prop that is not an empty string.` Le composant `SelectItem` de Radix UI ne supporte pas les valeurs vides (`""`) car cela entre en conflit avec le mecanisme de placeholder/clearing de Select.

## Stack trace
```

    at SelectItem (select.tsx:1278:13)
    at Object.react_stack_bottom_frame (react-dom-client.development.js:28038:20)
    at renderWithHooks (react-dom-client.development.js:7984:22)
    at updateForwardRef (react-dom-client.development.js:10059:19)
    at beginWork (react-dom-client.development.js:12475:18)
```

## Etapes de reproduction
1. Ouvrir la page /releves/nouveau
2. Selectionner un type de releve et une vague qui ont des activites planifiees
3. Observer l'erreur dans la console developpeur

## Cause racine
Story 13.5 a ajoute un `<Select>` pour lier une activite planifiee dans `releve-form-client.tsx`. Un des `<SelectItem>` a probablement une valeur vide (`""`) — soit un item de liste avec un id vide, soit un item "Aucune" dont la valeur est `""`.

## Fix
- [x] Fichier(s) modifie(s): `src/components/releves/releve-form-client.tsx`
- [x] Remplace `value=""` par `value="__auto__"` sur SelectItem (ligne 317)
- [x] Ajoute conversion `"__auto__"` <-> `""` dans value/onValueChange du Select
- [ ] Test de non-regression ajoute
- [ ] Tous les tests passent
- [ ] Build OK
