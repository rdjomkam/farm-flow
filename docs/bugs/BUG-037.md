# BUG-037 — VaguesComparison : fetch sans état de chargement
**Sévérité :** Moyenne
**Détecté par :** @architect
**Sprint :** UX
**Fichier(s) :** src/components/analytics/vagues-comparison-client.tsx

## Description
Le bouton "Comparer" dans la page de comparaison des vagues lance un fetch API sans aucun indicateur de chargement. Le bouton reste cliquable pendant le fetch, pas de spinner, pas de disabled.

## Étapes de reproduction
1. Aller sur /analytics/vagues
2. Sélectionner des vagues à comparer
3. Cliquer "Comparer"
4. Observer : aucun feedback, le bouton reste normal, les résultats apparaissent soudainement

## Cause racine
`handleComparer()` (lignes 428-438) appelle `analyticsService.getVagues()` sans état `isComparing`. Pas de `disabled` sur le bouton, pas de skeleton dans la zone résultats.

## Fix
- Ajouter état `isComparing` avec `useState`
- Désactiver le bouton pendant le fetch
- Afficher spinner/skeleton dans la zone résultats
- Voir plan détaillé : `docs/decisions/PLAN-ux-filters-loading.md` §P4
