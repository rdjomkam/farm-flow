# BUG-036 — BesoinsListClient : filtrage par onglet cassé (bug silencieux)
**Sévérité :** Haute
**Détecté par :** @architect
**Sprint :** UX
**Fichier(s) :** src/components/besoins/besoins-list-client.tsx

## Description
Tous les `TabsContent` de la liste des besoins affichent la même variable `filteredListes` calculée depuis `activeTab`. Quand l'utilisateur change d'onglet, le contenu affiché est toujours filtré par l'onglet actif (correct) MAIS les autres onglets montrent aussi les mêmes données. Radix Tabs masque le problème visuellement en cachant les onglets inactifs.

## Étapes de reproduction
1. Aller sur /besoins
2. S'assurer d'avoir des besoins dans différents statuts (soumises, approuvées, etc.)
3. Cliquer sur l'onglet "Soumises"
4. Observer le contenu (correct visuellement grâce à Radix)
5. Le bug est silencieux — il se manifeste si Radix ne cache pas correctement un onglet inactif

## Cause racine
`filteredListes` est calculé une seule fois depuis `activeTab` (lignes 120-127). Chaque `TabsContent` reçoit cette même variable. Pattern C1 mal implémenté — devrait utiliser Pattern C2 (un seul `TabsContent` avec `value={activeTab}`).

## Fix
- Remplacer les multiples `TabsContent` par un seul `<TabsContent value={activeTab}>`
- Ou calculer un array filtré différent pour chaque `TabsContent`
- Voir plan détaillé : `docs/decisions/PLAN-ux-filters-loading.md` §P1
