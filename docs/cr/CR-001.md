# CR-001 — Refonte navigation : menu hamburger + navigation par module
**Type :** Change Request (pas un bug)
**Severite :** Moyenne (UX structurelle, pas bloquant)
**Demande par :** Utilisateur via @team-lead
**Sprint prevu :** 12 (Story 12.6 — Reorganisation navigation)
**Fichier(s) :** src/components/layout/bottom-nav.tsx, sidebar.tsx, app-shell.tsx, nouveau composant hamburger

## Description
L'utilisateur demande une reorganisation complete de la navigation en **modules** avec un **menu hamburger**.

### Concept
1. **Menu hamburger** (trois barres) — navigation globale entre modules et pages secondaires (Sites, parametres, etc.)
2. **Bottom nav contextuelle par module** — change selon le module actif :
   - **Module "Suivi"** (/, /vagues, /releves, /bacs) : bottom nav = Dashboard, Vagues, Releve, Bacs
   - **Module "Stock"** (/stock/*) : bottom nav = Dashboard stock, Produits, Commandes, Mouvements, etc.
3. **Sites** — deplace dans le menu hamburger (plus dans la bottom nav)
4. **Stock** — accessible via le menu hamburger comme module, avec sa propre bottom nav

### Impact technique
- `src/components/layout/bottom-nav.tsx` — bottom nav dynamique selon le module actif (basee sur le pathname)
- `src/components/layout/sidebar.tsx` — potentiellement restructure en groupes par module
- Nouveau composant : menu hamburger (drawer/sheet mobile avec Radix Dialog ou Sheet)
- `src/components/layout/app-shell.tsx` — integrer le hamburger dans le header

### Relation avec le backlog existant
Cette CR enrichit la **Story 12.6 — Reorganisation navigation** deja prevue dans le Sprint 12.
La Story 12.6 actuelle prevoit :
- Mobile bottom-nav (5 items) avec sous-menus
- Desktop sidebar (groupes)
- Sous-menus animes

La CR-001 affine cette story avec le concept de modules + hamburger.

## Decision
Implementee en avance lors de la Phase 2 (post-Sprint 9). La navigation modulaire est en place :

### Implementation realisee
1. **Bottom nav contextuelle** (`bottom-nav.tsx`) — 4 contextes dynamiques :
   - Suivi : Dashboard, Vagues, Releve, Bacs, Analytiques
   - Stock : Stock, Produits, Fournisseurs, Commandes, Mouvements
   - Ventes : Clients, Ventes, Nouvelle, Factures
   - Detection automatique par pathname
2. **Sidebar modulaire** (`sidebar.tsx`) — 4 modules (Suivi, Analytiques, Stock, Ventes) + section secondaire (Sites)
3. **Hamburger menu complet** (`hamburger-menu.tsx`) — 4 modules avec sous-items, SiteSelector, section secondaire
4. **SiteSelector** dans hamburger + sidebar (pas dans header — CR-002)
