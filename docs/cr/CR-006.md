## Probleme

Trois ressources metier (Bac, Vague, Releve) ne peuvent pas etre modifiees apres creation, ou de maniere tres limitee. L'utilisateur doit supprimer et recreer la ressource pour corriger une erreur de saisie.

| Ressource | Etat actuel | Manque |
|-----------|-------------|--------|
| **Bac** | CREATE + LIST uniquement | Pas de PUT, pas de query `updateBac`, pas de `UpdateBacDTO`, pas d'UI edit |
| **Vague** | PUT existe mais limite a `statut`, `dateFin`, `addBacIds`, `removeBacIds` | Impossible de modifier `code`, `nombreInitial`, `poidsMoyenInitial`, `origineAlevins` |
| **Releve** | CREATE + LIST uniquement | Pas de PUT, pas de query `updateReleve`, pas de `UpdateReleveDTO`, pas d'UI edit |

### Contrainte d'acces

Seuls les utilisateurs disposant de la permission de modification correspondante peuvent mettre a jour :

| Ressource | Permission requise | Etat |
|-----------|-------------------|------|
| Bac | `BACS_MODIFIER` | **A creer** (actuellement seul `BACS_GERER` existe) |
| Vague | `VAGUES_MODIFIER` | Existe deja, utilise par `PUT /api/vagues/[id]` |
| Releve | `RELEVES_MODIFIER` | **A creer** |

## Regles metier

1. **Vague TERMINEE** : les champs `nombreInitial`, `poidsMoyenInitial`, `origineAlevins` ne doivent plus etre modifiables une fois la vague cloturee
2. **Bac assigne** : le `volume` peut etre modifie meme si un bac est assigne a une vague
3. **Releve** : le `typeReleve` ne peut jamais etre change (les champs dependant du type changeraient)
4. **Releve avec consommations** : si un releve a des `ReleveConsommation` liees, la mise a jour des quantites (`quantiteAliment`) ne modifie PAS les mouvements de stock existants (les consommations sont immutables)
5. **Permission stricte** : sans la permission `*_MODIFIER`, le PUT retourne 403

## Retrocompatibilite

- **100% retrocompatible** : ajout de nouveaux endpoints et permissions, aucun changement aux existants
- Les utilisateurs existants (ADMIN, GERANT) obtiennent automatiquement les nouvelles permissions via `DEFAULT_PERMISSIONS`
- Les PISCICULTEUR n'ont pas les permissions de modification par defaut (coherent avec leur role)
