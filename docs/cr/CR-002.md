# CR-002 — Déplacer le SiteSelector du Header vers le Menu Hamburger / Sidebar

**Type :** Change Request
**Sévérité :** Moyenne
**Demandé par :** @ronald
**Date :** 2026-03-09
**Sprint cible :** À planifier par @project-manager

## Problème

Le dropdown `SiteSelector` dans le header occupe ~110px sur mobile (360px), soit ~30% de la largeur du header. Sur les pages avec des boutons d'action (ex. `/vagues/[id]` avec "Relevé" + "Clôturer"), le header déborde ou compresse trop le titre. Le changement de site n'est pas assez fréquent pour justifier une place permanente dans le header.

## Changement proposé

Déplacer le sélecteur de site du header vers :
- **Mobile** : menu hamburger (drawer), section "Site actif" entre le logo et la navigation
- **Desktop** : sidebar, même emplacement

Le supprimer entièrement du header. Le header ne contiendra plus que : HamburgerMenu + Titre + children (boutons d'action) + UserMenu.

## Fichiers impactés

| Fichier | Modification |
|---------|-------------|
| `src/components/layout/site-selector.tsx` | Ajouter prop optionnelle `fullWidth` pour le contexte drawer/sidebar |
| `src/components/layout/header.tsx` | Supprimer le SiteSelector (instances mobile et desktop) + supprimer l'import |
| `src/components/layout/hamburger-menu.tsx` | Importer SiteSelector, ajouter section "Site actif" après le logo header |
| `src/components/layout/sidebar.tsx` | Importer SiteSelector, ajouter section "Site actif" après le logo header |

## Détail des changements

### 1. `site-selector.tsx` — Prop `fullWidth`

- Ajouter `interface SiteSelectorProps { fullWidth?: boolean }`
- Si `fullWidth` : conteneur `w-full`, bouton `w-full`, span `flex-1 text-left` (pas de `max-w-[80px]`)
- Si non `fullWidth` : comportement actuel conservé (compact avec troncature)

### 2. `header.tsx` — Suppression

- Supprimer l'import `SiteSelector`
- Supprimer les deux rendus : `<div className="hidden sm:block"><SiteSelector /></div>` et `<div className="sm:hidden"><SiteSelector /></div>`
- Résultat : plus d'espace pour le titre et les boutons d'action

### 3. `hamburger-menu.tsx` — Ajout section "Site actif"

- Importer `SiteSelector`
- Ajouter entre le logo header et `<nav>` :
  - Label "SITE ACTIF" (style `text-xs uppercase tracking-wider text-muted-foreground`)
  - `<SiteSelector fullWidth />`
  - Séparateur `border-b`

### 4. `sidebar.tsx` — Ajout section "Site actif"

- Même pattern que le hamburger menu
- Section entre le logo et `<nav>`

## Impact

- **UX positive** : ~30% d'espace header récupéré, plus de débordement sur les pages avec actions
- **UX mineure négative** : changement de site nécessite 1 tap supplémentaire (ouvrir le menu)
- **Risque** : faible — changement purement layout/CSS, pas de logique métier modifiée
- **Tests** : aucun test existant impacté (les tests header ne vérifient pas le SiteSelector)

## Vérification post-implémentation

1. `npm run build` — Build OK

3. Menu hamburger : SiteSelector visible en haut, changement de site fonctionne
4. Sidebar desktop : SiteSelector visible en haut, changement de site fonctionne
5. Lien "Gérer les sites" toujours accessible dans les items secondaires
6. `npx vitest run` — Tous les tests passent
