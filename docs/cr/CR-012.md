# CR-012 — Module Analytiques : sous-pages et types d'analyses supplementaires

**Type :** Change Request
**Severite :** Haute
**Demande par :** @ronald
**Date :** 2026-03-10
**Sprint cible :** Sprint 11-12
**Depend de :** CR-010 (analytiques bacs), CR-011 (analytiques aliments), Sprint 11 (finances)

## Contexte

Le module "Analytiques" existe deja dans la navigation (hamburger, sidebar, bottom-nav) avec 2 sous-pages :
- `/analytics/bacs` — Comparaison par bac (CR-010)
- `/analytics/aliments` — Comparaison par aliment (CR-011)

Pour positionner Farm-Flow comme un SaaS premium en pisciculture, le module Analytiques doit devenir le coeur de l'application — la raison pour laquelle un pisciculteur paie un abonnement plutot que d'utiliser un cahier ou Excel.

Ce CR definit les types d'analyses supplementaires a ajouter comme sous-pages du module.

## Sous-pages du module Analytiques

### Vue d'ensemble de la structure

```
Analytiques (module)
├── /analytics                   → Dashboard analytique (vue globale)
├── /analytics/bacs              → Comparaison par bac (CR-010, FAIT)

├── /analytics/finances          → Analyse financiere (apres Sprint 11)

```

### 2. Comparaison entre vagues (`/analytics/vagues`)

**Objectif :** Comparer les performances de differentes vagues/cycles pour identifier les meilleures pratiques et les regressions.

**Metriques par vague :**

| Metrique | Formule | Pourquoi |
|----------|---------|----------|
| FCR global | totalAliment / gainBiomasse | Efficacite alimentaire du cycle |
| Taux de survie final | dernierComptage / nombreAlevins × 100 | Resultat sante du cycle |
| SGR moyen | Moyenne ponderee des SGR releves | Vitesse de croissance |
| Duree du cycle | dateFin - dateDebut (jours) | Temps de retour sur investissement |
| Biomasse recoltee | Estimation ou vente reelle | Production totale |
| Cout total aliment | SUM(consommations × prix) | Depense alimentaire |
| Cout par kg produit | coutAliment / biomasseProduite | Rentabilite alimentaire |
| Revenu de vente | SUM(Vente.montantTotal) pour cette vague | Chiffre d'affaires |
| Marge brute | revenuVentes - coutAliment | Profit sur le cycle |
| ROI | (revenuVentes - coutTotal) / coutTotal × 100 | Rentabilite globale |

**UI :**
- Selection de 2-4 vagues a comparer (dropdown multi-select)
- Cartes empilees sur mobile avec metriques cote a cote
- Graphique radar (Recharts) pour visualiser le profil de chaque vague
- Graphique en barres groupees pour comparer les metriques financieres
- Code couleur : meilleure vague en vert, pire en rouge

**Donnees necessaires :**
- `Vague` — deja disponible
- `Releve` agrege par vague — deja disponible (cf. `getIndicateursVague`)
- `Vente` par vague — disponible depuis Sprint 9 (`Vente.vagueId`)
- `ReleveConsommation` par vague — disponible

**Fichiers :**
- `src/app/analytics/vagues/page.tsx` — Page comparaison vagues
- `src/components/analytics/vague-comparison-cards.tsx` — Cartes comparaison
- `src/components/analytics/vague-radar-chart.tsx` — Graphique radar Recharts
- `src/lib/queries/analytics.ts` — Ajouter `getComparaisonVagues(siteId, vagueIds[])`
- `src/types/calculs.ts` — Ajouter `ComparaisonVagues`, `IndicateursVague` (etendu)

### 3. Analyse financiere (`/analytics/finances`)

**Objectif :** Donner une vue financiere complete de l'exploitation. C'est le tableau de bord que le gerant montre a son banquier ou investisseur.

> **Note :** Ce CR definit la page analytics/finances. Sprint 11 (Story 11.4 + 11.7 + 11.10) definit `/finances` comme dashboard financier operationnel. Les deux peuvent coexister ou etre fusionnes — decision a prendre lors de l'implementation. La page `/analytics/finances` se concentre sur l'analyse comparative et les tendances, pas sur le suivi quotidien.

**Metriques :**

| Metrique | Formule | Source |
|----------|---------|--------|
| Revenus totaux | SUM(Vente.montantTotal) | Vente (Sprint 9) |
| Couts aliments | SUM(MouvementStock.prixTotal) WHERE type=ENTREE, categorie=ALIMENT | MouvementStock + Produit |
| Couts intrants | SUM(MouvementStock.prixTotal) WHERE categorie=INTRANT | MouvementStock + Produit |
| Marge brute | Revenus - CoutsAliments - CoutsIntrants | Calcul |
| Taux de marge | MargeBrute / Revenus × 100 | Calcul |
| Encaissements | SUM(Paiement.montant) | Paiement (Sprint 9) |
| Creances | Revenus - Encaissements | Calcul |
| Rentabilite par vague | (Vente.montantTotal - couts) / couts × 100 | Croise |
| Prix moyen de vente | SUM(montantTotal) / SUM(poidsTotalKg) | Vente |
| Cout moyen de production | coutsTotaux / biomasseRecoltee | Croise |

**UI :**
- Filtres par periode (mois, trimestre, annee, personnalise)
- Cartes KPI : revenus, couts, marge, taux de marge
- Graphique courbe : evolution revenus vs couts par mois (Recharts AreaChart)
- Graphique barres : rentabilite par vague (du plus au moins rentable)
- Graphique camembert : repartition des couts (aliment, intrants, equipement)
- Tableau simplifie : top 5 clients par chiffre d'affaires

**Depend de :** Sprint 11 (Story 11.4 — queries finances). Reutiliser `src/lib/queries/finances.ts` une fois cree.

**Fichiers :**
- `src/app/analytics/finances/page.tsx` — Page analyse financiere
- `src/components/analytics/finance-kpi-cards.tsx` — Cartes KPI financieres
- `src/components/analytics/revenue-cost-chart.tsx` — Courbe revenus vs couts
- `src/components/analytics/profitability-chart.tsx` — Barres rentabilite par vague
- `src/components/analytics/cost-breakdown-chart.tsx` — Camembert repartition couts
- `src/lib/queries/analytics.ts` — Ajouter `getAnalytiqueFinanciere(siteId, periode)`
- `src/types/calculs.ts` — Ajouter `AnalytiqueFinanciere`, `RentabiliteVague`

### 4. Tendances et previsions (`/analytics/tendances`)

**Objectif :** Identifier les tendances sur le long terme et projeter les performances futures. C'est la page la plus "SaaS premium" — elle justifie l'abonnement.

**Analyses :**

| Analyse | Description | Source |
|---------|-------------|--------|
| Evolution FCR | Courbe FCR moyen sur 6-12 mois. Tendance ascendante = degradation, descendante = amelioration | Releves ALIMENTATION + BIOMETRIE |
| Evolution survie | Courbe taux de survie par cycle. Detecte si la mortalite augmente | Releves MORTALITE + COMPTAGE |
| Saisonnalite | Comparer les memes mois d'une annee sur l'autre (FCR en saison des pluies vs saison seche) | Releves + dates |
| Prevision recolte | Projeter la date et le poids de recolte en fonction du SGR actuel | Releves BIOMETRIE recents |
| Prevision cout aliment | Estimer le cout aliment restant pour la vague en cours | Consommation actuelle + FCR + biomasse cible |
| Benchmark site | Comparer les performances du site aux benchmarks Clarias gariepinus (cf. CR-010 benchmarks) | Constantes benchmarks + donnees site |

**UI :**
- Graphique principal : courbe multi-serie (FCR, survie, SGR) sur 12 mois
- Section "Previsions" avec cartes : date de recolte estimee, biomasse estimee, cout restant estime
- Section "Benchmark" : jauges montrant ou le site se situe par rapport aux standards (vert/jaune/rouge)
- Pas de tableaux — cartes + graphiques uniquement

**Fichiers :**
- `src/app/analytics/tendances/page.tsx` — Page tendances
- `src/components/analytics/trend-chart.tsx` — Graphique multi-serie Recharts
- `src/components/analytics/forecast-cards.tsx` — Cartes prevision
- `src/components/analytics/benchmark-gauges.tsx` — Jauges benchmark
- `src/lib/calculs.ts` — Ajouter `projeterRecolte(sgr, poidsMoyen, poidsCible)`, `estimerCoutRestant(fcrActuel, biomasseCible, biomasseActuelle, prixAliment)`
- `src/lib/queries/analytics.ts` — Ajouter `getTendances(siteId, mois)`, `getPrevisions(siteId, vagueId)`

## Argument SaaS

Chaque sous-page repond a une question business que le pisciculteur pose :

| Question | Sous-page |
|----------|-----------|
| "Quel bac performe le mieux ?" | Par bac |
| "Quel aliment est le plus rentable ?" | Par aliment |
| "Ma derniere vague etait-elle meilleure que la precedente ?" | Par vague |
| "Mon exploitation est-elle rentable ?" | Finances |
| "Est-ce que mes performances s'ameliorent ou se degradent ?" | Tendances |
| "Ou en suis-je globalement ?" | Dashboard |

Un concurrent qui ne repond qu'a 1-2 de ces questions n'est pas un vrai outil de gestion — c'est un carnet glorifie. Farm-Flow repond aux 6.
