# CR-008 — Repenser les actions sur les membres de site (dialog ou page dediee)

**Type :** Change Request
**Severite :** Moyenne
**Demande par :** @ronald
**Date :** 2026-03-09
**Sprint cible :** Immediat

## Probleme

Les actions sur un membre de site (changer le role, gerer les permissions, retirer) sont actuellement affichees inline sous chaque membre dans la liste. Sur mobile 360px, cela pose plusieurs problemes :

1. **Encombrement** : le select role + 2 boutons icones (permissions, retirer) sur une ligne de 280px utiles est serre
2. **Risque d'erreur** : le bouton Retirer est a cote du bouton Permissions — un tap accidentel peut retirer un membre
3. **Pas de confirmation** : la suppression se fait en un seul tap sans confirmation
4. **Navigation cachee** : le bouton Shield (permissions) mene a une page separee sans indication claire

## Proposition

Remplacer les actions inline par une approche dialog/page par membre :

### Option retenue : Dialog de gestion membre

Quand l'utilisateur tape sur un membre (ou un bouton "Gerer"), ouvrir un dialog avec :

```
+---------------------------+
| Gerer — Jean Kamga    [X] |
+---------------------------+
|                           |
|  Email: gerant@dkfarm.cm  |
|  Tel: +237 677 000 000    |
|                           |
|  ROLE                     |
|  [Gerant            v]   |
|                           |
|  PERMISSIONS              |
|  [Gerer les permissions >]|
|                           |
|  ─────────────────────    |
|  [Retirer du site]        |  <- rouge, avec confirmation
|                           |
+---------------------------+
```

### Avantages
- **Plus d'espace** : chaque action a sa propre ligne
- **Confirmation** : la suppression demande confirmation dans le dialog
- **Decouverte** : les permissions sont clairement affichees comme option
- **Mobile-friendly** : dialog standard, pas de micro-boutons

## Fichiers impactes

| Fichier | Modification |
|---------|-------------|
| `src/components/sites/site-detail-client.tsx` | Remplacer les actions inline par un bouton "Gerer" (ou tap sur le membre) qui ouvre un dialog |
| Nouveau : `src/components/sites/member-actions-dialog.tsx` | Dialog avec : info contact, select role, lien permissions, bouton retirer avec confirmation |

## Regles

- Le dialog n'apparait que pour les membres que l'utilisateur peut modifier (canModify)
- Pour les membres non modifiables (soi-meme, admin si non-admin), afficher le badge role en lecture seule
- La suppression doit demander confirmation ("Retirer [Nom] du site ?")
- Le lien permissions navigue vers `/sites/[id]/members/[userId]/permissions` (page existante)

## Verification

1. `npm run build` OK
2. Mobile 360px : liste membres plus propre, actions accessibles via dialog
3. Confirmation avant suppression
4. Pas de regression sur les fonctionnalites existantes (ajout, role, permissions, retrait)
